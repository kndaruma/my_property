# 🏠 MyProperty - Advanced Property System

ยินดีต้อนรับสู่สคริปต์ระบบบ้าน (Property System) ที่ได้รับการปรับปรุงและ Optimize ให้ทำงานได้อย่างมีประสิทธิภาพสูงสุด รองรับการสร้างบ้านไม่จำกัด, ระบบเฟอร์นิเจอร์, ระบบกุญแจ, ลิฟต์มิติซ้อน, ระบบปล่อยเช่า และระบบ Donate

---

## ⚠️ สิ่งสำคัญมาก (ต้องทำก่อนเริ่มใช้งาน) ⚠️

เพื่อให้ระบบจัดการประตูบ้าน (Dynamic Doors) และการตรวจจับโมเดลทำงานได้อย่างสมบูรณ์ คุณ **ต้อง** นำคำสั่งด้านล่างนี้ไปใส่ไว้ในไฟล์ 
`server.cfg` ของเซิร์ฟเวอร์คุณ (แนะนำให้ใส่ไว้ด้านบนสุด หรือก่อนคำสั่ง `ensure` สคริปต์นี้):

setr game_enableDynamicDoorCreation "true"

และสร้าง Folder Backup_Property ด้วย

## 🌟 ฟีเจอร์หลักของสคริปต์ (Core Features)
`Dimesion System (แยกมิติ):` ผู้เล่นแต่ละหลังจะอยู่ในมิติ `(Routing Bucket)` ของตัวเอง ทำให้สามารถสร้างบ้านทับที่กันได้โดยไม่รบกวนกัน

`Furniture System:` ระบบซื้อ วาง ขยับ คัดลอก และขายเฟอร์นิเจอร์ พร้อมระบบค้นหาแบบพิมพ์ชื่อ `(/sf)`

`Dynamic Doors (ประตูอัจฉริยะ):` ระบบตรวจสอบโมเดลประตูอัตโนมัติ (Whitelist ใน config.lua) ประตูจริงจะเดินชนเปิดได้ ส่วน Prop ปลอมจะถูกแช่แข็งเป็นกำแพงทันที ป้องกันปัญหาเดินชนแล้วล้ม

`Freecam Placement (/pe):` โหมดบินอิสระ ช่วยให้ผู้เล่นจัดวางเฟอร์นิเจอร์ในที่สูงหรือมุมอับได้ง่ายขึ้น

`Elevator/Lift System (/sl):` ระบบลิฟต์ มิติซ้อนมิติ สำหรับบ้านที่มีหลายชั้น

`Access Management:` ระบบเคาะประตู `(/knockd)`, ล็อคประตู `(/ld)`, เชิญคนเข้าบ้าน `(/invitetop)` และมอบกุญแจให้เพื่อน

`Property Market:` ผู้เล่นสามารถประกาศขายบ้านของตัวเองให้ผู้เล่นคนอื่นได้โดยตรง

`Rent System:` ระบบปล่อยเช่าบ้านแบบรายวัน ผู้เช่ามีสิทธิ์เทียบเท่าคนถือกุญแจ พร้อมระบบแจ้งเตือนอัตโนมัติเมื่อใกล้หมดสัญญาเช่า

`Donate Feature:` ระบบตั้งเวลาและสภาพอากาศส่วนตัวภายในบ้าน เฉพาะบ้านที่มีสถานะ Donate เท่านั้น

`Auto-Backup:` ระบบสำรองข้อมูลบ้านแต่ละหลังเป็นไฟล์ .json อัตโนมัติทุกๆ 2 ชั่วโมง

## 📁 โฟลเดอร์หลัก (Folders)
`Backup_property/`
    หน้าที่: โฟลเดอร์สำหรับเก็บไฟล์ สำรองข้อมูล (Backup) ของบ้านแต่ละหลัง ระบบจะสร้างไฟล์ .json ให้อัตโนมัติ ป้องกันข้อมูลหายหาก Database มีปัญหา

`client/`
    หน้าที่: โฟลเดอร์เก็บสคริปต์ทั้งหมดที่ทำงานฝั่งผู้เล่น (Client-side)

`RageUI/`
    หน้าที่: ไลบรารี (Library) สำเร็จรูปที่ใช้สำหรับการสร้างหน้าต่างเมนู UI ที่สวยงามและลื่นไหล

`server/`
    หน้าที่: โฟลเดอร์เก็บสคริปต์ทั้งหมดที่ทำงานฝั่งเซิร์ฟเวอร์และฐานข้อมูล (Server-side)

## 📄 ไฟล์ต่างๆ (Files)
`client/client.lua`
    ไฟล์หลักของฝั่งผู้เล่น จัดการเรื่องการแสดงผลเมนู RageUI ทั้งหมด, ระบบตรวจจับพิกัด, การสร้างจุดวาร์ป (Marker), การจัดการกุญแจ, ประตู และดึงข้อมูลบ้าน

`client/client_lift.lua`
    จัดการระบบลิฟต์ (Elevator) โดยเฉพาะ ดูแลเรื่องเมนูลิฟต์ การสร้างจุดวาร์ปข้ามชั้น และมิติย่อยภายในบ้าน

`client/client_pedit.lua`
    ควบคุมระบบ Property Editor (/pe) โหมดบินอิสระ จัดการปุ่มกด (WASD) และการตั้งค่าความเร็ว

`client/placement.lua`
    ควบคุมการจัดวางเฟอร์นิเจอร์ (Placement Mode) แกน X, Y, Z, องศา (Rotation) และการตั้งค่า Freeze Object

`server/server.lua`
    ไฟล์หัวใจหลักของเซิร์ฟเวอร์ ทำหน้าที่ติดต่อฐานข้อมูล ประมวลผลคำสั่ง หักเงิน/เพิ่มเงิน ตรวจสอบสิทธิ์ผู้เล่น และรันระบบ Auto-Backup

`Commands Documentation.md`
    คู่มือรวมคำสั่ง (Commands) ทั้งหมดที่มีในสคริปต์ แบ่งหมวดหมู่สิทธิ์การใช้งาน

`config.lua`
    ไฟล์ตั้งค่าหลักของสคริปต์ ใช้ตั้งค่า Dynamic Doors, แก้ไขสีจุดวาร์ป และ จัดการร้านค้าเฟอร์นิเจอร์

`database.sql`
    ไฟล์โครงสร้างฐานข้อมูล (Database) ที่ใช้สร้างตาราง my_properties และ my_property_furniture ใน Database

    โปรดอ่านไฟล์ Commands Documentation.md เพื่อดูวิธีใช้คำสั่งต่างๆ ในสคริปต์นี้อย่างละเอียด
