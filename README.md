# 🏠 MyProperty - Advanced Property System

ยินดีต้อนรับสู่สคริปต์ระบบบ้าน (Property System) ที่ได้รับการปรับปรุงและ Optimize ให้ทำงานได้อย่างมีประสิทธิภาพสูงสุด รองรับการสร้างบ้านไม่จำกัด, ระบบเฟอร์นิเจอร์, ระบบกุญแจ, ลิฟต์มิติซ้อน และระบบ Donate

---

## ⚠️ สิ่งสำคัญมาก (ต้องทำก่อนเริ่มใช้งาน) ⚠️

เพื่อให้ระบบจัดการประตูบ้าน (Dynamic Doors) และการตรวจจับโมเดลทำงานได้อย่างสมบูรณ์ คุณ **ต้อง** นำคำสั่งด้านล่างนี้ไปใส่ไว้ในไฟล์ 
`server.cfg` ของเซิร์ฟเวอร์คุณ (แนะนำให้ใส่ไว้ด้านบนสุด หรือก่อนคำสั่ง `ensure` สคริปต์นี้):

setr game_enableDynamicDoorCreation "true"

## 🌟 ฟีเจอร์หลักของสคริปต์ (Core Features)
`Dimesion System (แยกมิติ):` ผู้เล่นแต่ละหลังจะอยู่ในมิติ `(Routing Bucket)` ของตัวเอง ทำให้สามารถสร้างบ้านทับที่กันได้โดยไม่รบกวนกัน

`Furniture System:` ระบบซื้อ วาง ขยับ คัดลอก และขายเฟอร์นิเจอร์ พร้อมระบบค้นหาแบบพิมพ์ชื่อ `(/sf)`

`Freecam Placement (/pe):` โหมดบินอิสระ ช่วยให้ผู้เล่นจัดวางเฟอร์นิเจอร์ในที่สูงหรือมุมอับได้ง่ายขึ้น

`Elevator/Lift System (/sl):` ระบบลิฟต์ มิติซ้อนมิติ สำหรับบ้านที่มีหลายชั้น

`Access Management:` ระบบเคาะประตู `(/knockd)`, ล็อคประตู `(/ld)`, เชิญคนเข้าบ้าน `(/invitetop)` และมอบกุญแจให้เพื่อน

`Property Market:` ผู้เล่นสามารถประกาศขายบ้านของตัวเองให้ผู้เล่นคนอื่นได้โดยตรง

`Donate Feature:` ระบบตั้งเวลาและสภาพอากาศส่วนตัวภายในบ้าน เฉพาะบ้านที่มีสถานะ Donate เท่านั้น

`Auto-Backup:` ระบบสำรองข้อมูลบ้านแต่ละหลังเป็นไฟล์ .json อัตโนมัติทุกๆ 2 ชั่วโมง

## 📁 โฟลเดอร์หลัก (Folders)
`Backup_property/`
    หน้าที่: โฟลเดอร์สำหรับเก็บไฟล์ สำรองข้อมูล (Backup) ของบ้านแต่ละหลัง ระบบจะสร้างไฟล์ .json (เช่น Backup_Property_1.json) ให้อัตโนมัติเมื่อมีการสร้างบ้านใหม่ หรือถึงรอบเวลาเซฟ ป้องกันข้อมูลหายหาก Database มีปัญหา

`client/`
    หน้าที่: โฟลเดอร์เก็บสคริปต์ทั้งหมดที่ทำงานฝั่งผู้เล่น (Client-side)

`RageUI/`
    หน้าที่: ไลบรารี (Library) สำเร็จรูปที่ใช้สำหรับการสร้างหน้าต่างเมนู UI ที่สวยงามและลื่นไหลภายในสคริปต์นี้

`server/`
    หน้าที่: โฟลเดอร์เก็บสคริปต์ทั้งหมดที่ทำงานฝั่งเซิร์ฟเวอร์และฐานข้อมูล (Server-side)

## 📄 ไฟล์ต่างๆ (Files)
`client/client.lua`
    ไฟล์หลักของฝั่งผู้เล่น จัดการเรื่องการแสดงผลเมนู RageUI ทั้งหมด, ระบบตรวจจับพิกัด, การสร้างจุดวาร์ป (Marker), การจัดการกุญแจ, ประตู และดึงข้อมูลบ้านมาแสดงผลบนหน้าจอ

`client/client_lift.lua`
    จัดการระบบลิฟต์ (Elevator) โดยเฉพาะ ดูแลเรื่องเมนูลิฟต์ การสร้างจุดวาร์ปข้ามชั้น และการคำนวณมิติย่อยภายในบ้านหลังเดียวกัน

`client/client_pedit.lua`
    ควบคุมระบบ Property Editor (/pe) หรือโหมดบินอิสระ จัดการปุ่มกด (WASD) การตั้งค่าความเร็วในการบิน และการปิดบังคับแอนิเมชั่นตัวละคร

`client/placement.lua`
    ควบคุมการจัดวางเฟอร์นิเจอร์ (Placement Mode) แกน X, Y, Z, องศา (Rotation), ความรวดเร็วในการเลื่อนวัตถุ และการเรนเดอร์ UI ช่วยเหลือด้านซ้ายของจอ

`server/server.lua`
    ไฟล์หัวใจหลักของเซิร์ฟเวอร์ ทำหน้าที่ติดต่อกับฐานข้อมูล (MySQL) ประมวลผลคำสั่งของแอดมิน (สร้างบ้าน, ตั้งช่อง, แจก Donate), ระบบตลาดซื้อขายบ้าน, หักเงิน/เพิ่มเงิน, ตรวจสอบสิทธิ์ผู้เล่น และรันระบบ Auto-Backup

`Commands Documentation.md`
    คู่มือรวมคำสั่ง (Commands) ทั้งหมดที่มีในสคริปต์ แบ่งหมวดหมู่สิทธิ์การใช้งาน (Admin, Owner, General) เอาไว้สำหรับแจกจ่ายให้ทีมงานหรือผู้เล่นอ่าน

`config.lua`
    ไฟล์ตั้งค่าหลักของสคริปต์ ใช้สำหรับแก้ไขสีจุดวาร์ป และ เพิ่ม/ลด/แก้ไข รายการเฟอร์นิเจอร์ รวมถึงราคาและหมวดหมู่ในร้านค้า

`database.sql`
    ไฟล์โครงสร้างฐานข้อมูล (Database) ที่ใช้สำหรับสร้างตาราง my_properties และ my_property_furniture ใน phpMyAdmin

`fxmanifest.lua`
    ไฟล์ประกาศข้อมูลของสคริปต์สำหรับ FiveM (ตัวบอกว่าสคริปต์ชื่ออะไร และต้องโหลดไฟล์ไหนบ้าง)
