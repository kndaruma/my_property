## ระบบ Property ที่อนุญาตให้แอดมินสร้างบ้านหรือพื้นที่ส่วนตัวให้ผู้เล่น โดยใช้ระบบมิติ (Routing Bucket) เพื่อแยกโลกออกจากกัน 
## ผู้ถือครองกุญแจสามารถบริหารจัดการบ้านได้เต็มรูปแบบ เช่น ซื้อเฟอร์นิเจอร์, จัดวางของ, สร้างลิฟต์หลายชั้น, ล็อคประตูบ้าน, และตั้งเวลาเฉพาะภายในบ้านได้

!!!!!!!!!!!!!!!!!!!! ก่อนใช้ Script นี้ ต้อง Set ค่า setr game_enableDynamicDoorCreation "true" นี้ใน Server.cfg ใส่ก่อน ensure script ต่างๆของเซิฟเวอร์ก่อน เพื่อให้ ประตูที่มี Dynamic ใช้งานได้และขยับประตูได้ !!!!!!!!!!!!!!!!!!

RageUI/ (Folder) = โฟลเดอร์เก็บ Library สำเร็จรูปที่ใช้สำหรับสร้างหน้าต่างเมนูทั้งหมดในสคริปต์นี้

client.lua = สคริปต์หลักฝั่งผู้เล่น (Client-side) คุมระบบเกือบทั้งหมด เช่น การเปิดเมนูจัดการบ้าน, การโหลด/ลบเฟอร์นิเจอร์ตอนเดินเข้า-ออก, ระบบตั้งเวลาในบ้าน, ระบบปุ่มล็อคประตู และคืนค่าการชน (Collision)

client_lift.lua = สคริปต์จัดการระบบ "ลิฟต์ (Elevator)" คุมการสร้างชั้นใหม่, ลบชั้น, การเดินเข้าวงมาร์คเกอร์ลิฟต์ และการวาร์ปข้ามมิติย่อยในบ้านหลังเดียวกัน

client_pedit.lua = สคริปต์จัดการระบบ "Property Editor" หรือโหมดบิน (Freecam) สำหรับให้ผู้เล่นบินทะลุสิ่งกีดขวางเพื่อจัดวางเฟอร์นิเจอร์ได้ง่ายขึ้น

config.lua = ไฟล์ตั้งค่าหลัก ใช้สำหรับกำหนดรูปแบบจุดวาร์ป (Marker) โทนสีต่างๆ และรายการ/ราคาของเฟอร์นิเจอร์ทั้งหมดที่ขายในเมนู Buy Furniture

database.sql = ไฟล์คำสั่ง SQL สำหรับนำไปรันในฐานข้อมูล (Database) เพื่อสร้างตารางเก็บข้อมูลบ้าน (my_properties) และเฟอร์นิเจอร์ (my_property_furniture)

fxmanifest.lua = ไฟล์สารบัญของสคริปต์ (Resource Manifest) ทำหน้าที่บอก FiveM ว่าสคริปต์นี้ชื่ออะไร และต้องโหลดไฟล์ไหนก่อน-หลัง (Load Order)

placement.lua = สคริปต์คุมระบบ "จำลองการจัดวาง (Preview)" คุมปุ่มกดบังคับต่างๆ ตอนที่กำลังเลื่อนของ (WASD, ขึ้นลง, หมุน) และคุมสถานะการชนระหว่างจัดวาง

server.lua = สคริปต์ฝั่งเซิร์ฟเวอร์ (Server-side) ทำหน้าที่คุยกับฐานข้อมูล (SQL), ตรวจสอบสิทธิ์ (Admin/Keyholder), ซิงค์ข้อมูลอัปเดตไปให้ผู้เล่นทุกคน และคุมการย้ายมิติ (Routing Bucket)

# รายการคำสั่ง (Commands & Permissions)

# คำสั่งสำหรับผู้ดูแลระบบ (Admin Only)

/createhouse [จำนวนช่องเฟอร์นิเจอร์]
การทำงาน: สร้าง Property ใหม่ โดยดึงพิกัดจุดที่แอดมินยืนอยู่เป็น "ทางเข้า"

/setslots [ID] [จำนวน]
การทำงาน: เปลี่ยนจำนวนจำกัดชิ้น ชิ้นส่วนเฟอร์นิเจอร์สูงสุดของบ้านนั้นๆ

/deleteproperty [ID]
ต้องยืนใกล้จุดทางเข้าบ้านนั้นๆ ไม่เกิน 3 เมตร
การทำงาน: ลบ Property และเฟอร์นิเจอร์ทั้งหมดทิ้งถาวร (เตะคนที่อยู่ข้างในออกมาอัตโนมัติ)

# คำสั่งสำหรับเจ้าของ/ผู้ถือกุญแจ (Keyholder & Admin)

/manageproperty หรือตัวย่อ /mp
ภายในบ้าน เท่านั้น
การทำงาน: เปิดเมนูหลักสำหรับจัดการบ้าน (ประตู, เฟอร์นิเจอร์, ให้กุญแจผู้เล่นอื่น)

/buyfurniture หรือตัวย่อ /bf
ภายในบ้าน เท่านั้น
การทำงาน: เปิดเมนูร้านค้าเฟอร์นิเจอร์โดยตรง

/setexit [ชื่อประตู] หรือตัวย่อ /se [ชื่อประตู]
ภายในบ้าน (จุดที่อยากให้ออกมาโผล่)
การทำงาน: ตั้งค่าจุดเมื่อเดินเข้าประตูนั้นๆ มาจากข้างนอก หากใช้ /setexit เฉยๆจะเป็นการตั้งจุดของประตูหลัก แต่หาก /setexit ตามด้วยชื่อประตู ก็จะเป็นจุดของประตูนั้น

/setentrance [ID บ้าน] [ชื่อประตู]
ใช้ข้างนอก (โลกหลัก)
การทำงาน: ตั้งค่าจุดทางเข้าของประตูเสริมจากโลกภายนอก

/lockdoor หรือตัวย่อ /ld
ใช้ได้ทั้ง ในบ้าน หรือ นอกบ้าน (ต้องยืนทับวงวาร์ปประตูเท่านั้น)
การทำงาน: ล็อคประตู ไม่ให้คนอื่นเข้า/ออกได้ (มาร์คเกอร์จะเปลี่ยนเป็นสีแดงและล็อคแค่จุดที่ล็อคเอาไว้หากจุดอื่นไม่ล็อคก็สามารถเข้าไปได้)

/unlockdoor หรือตัวย่อ /ud
ใช้ได้ทั้ง ในบ้าน หรือ นอกบ้าน (ต้องยืนทับวงวาร์ปประตูเท่านั้น)
การทำงาน: ปลดล็อคประตู

/setptime [ชั่วโมง 0-23] หรือ /setptime reset
ภายในบ้าน เท่านั้น
การทำงาน: ล็อกเวลาตายตัวเฉพาะภายในบ้านหลังนั้น (พิมพ์ reset เพื่อคืนค่าให้ตรงกับเซิร์ฟเวอร์หลัก)

/setlift หรือตัวย่อ /sl
ภายในบ้าน เท่านั้น
การทำงาน: เปิดเมนูลิฟต์ หากบ้านยังไม่มีลิฟต์จะเป็นการสร้างจุดลิฟต์ชั้น 1 ที่จุดที่ยืนอยู่ และกด E ที่วงลิฟต์ จะสามารถสร้างชั้นอื่นๆได้

/peditmode หรือตัวย่อ /pe
ภายในบ้าน เท่านั้น
การทำงาน: เข้าโหมดวิญญาณ (Freecam) บินทะลุสิ่งของเพื่อความสะดวกในการจัดวางเฟอร์นิเจอร์

/invitetop
ภายในบ้าน เท่านั้น
การทำงาน: ใช้เชิญคนเข้าบ้านขณะที่ประตูล็อคอยู่ และต้องเลือก invite ที่ Manage Door และเลือกประตูนั้นก่อน

/sellproperty [ราคา] หรือไม่ก็ /sellproperty [ID Property] [ราคา] สักอันนึงจำไม่ได้ 555555555555
จุดวาร์ปหลักบ้าน
การทำงาน: ใช้ขาย Property สำหรับ Owner

# คำสั่งทั่วไป (Everyone)

/buyproperty
จุดวาร์ปหลักบ้าน
การทำงาน: ใช้ซื้อ Property สำหรับ Player

/checkhouse

การทำงาน: เช็คว่าประตูนี้คือ Property ID อะไร (มีประโยชน์ตอนแอดมินจะใช้คำสั่งจัดการ)

/dleave
ใช้ภายในบ้าน
การทำงาน: คำสั่งฉุกเฉินสำหรับผู้เล่นทั่วไป เอาไว้วาร์ปตัวเองออกจากมิติบ้านกลับไปโลกหลัก (เผื่อประตูล็อคหรือหาทางออกไม่เจอ)

/clearprops
ใช้ที่ไหนก็ได้
การทำงาน: ลบโมเดลเฟอร์นิเจอร์บัค ที่ค้างอยู่ในรัศมี 5 เมตรรอบตัวผู้เล่น

/checkdim
ใช้ที่ไหนก็ได้
การทำงาน: เช็คว่าตัวเองอยู่ในมิติ (Routing Bucket) หมายเลขอะไร

/knockd
ใช้ที่วงวาร์ปประตูนั้นๆ
การทำงาน: จะเคาะประตูแล้วจะมีแจ้งเตือนไปยังคนในบ้านที่ถือกุญแจทุกคน
